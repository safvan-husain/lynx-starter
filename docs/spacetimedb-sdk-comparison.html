SpacetimeDB SDK Comparison

This compares the local Lynx port used by this app, the Dart package, and the official SpacetimeDB React integration from the TypeScript SDK. The last column focuses on gaps that matter if spacetimedb-lynx is expected to support full frontend application behavior.

Functional comparison
Area Official React / TypeScript SDK Dart package Local spacetimedb-lynx Gap in spacetimedb-lynx
Package / source spacetimedb/react, official TypeScript SDK integration. spacetimedb on pub.dev for Dart/Flutter. Local package at spacetimedb-lynx/, consumed via file:../spacetimedb-lynx. Partial: local fork/port must be kept in sync with upstream SDK changes.
Target runtime Browser/Node TypeScript apps, with React, Vue, and Svelte integrations. Dart and Flutter apps across mobile, desktop, and web. Lynx runtime, with native WebSocket and HTTP adapters. Intentional difference: Lynx needs custom transport, but behavior should mirror official TS SDK.
Connection setup DbConnection.builder(), withUri, withDatabaseName, withToken, connect/disconnect callbacks. SpacetimeDbClient.connect(...) with host, database, SSL, auth storage, and initial subscriptions. Uses the same builder API plus withWS(LynxWebSocketAdapter) and withFetchFn(lynxFetch). Mostly covered.
Transport WebSocket protocol v2.bsatn.spacetimedb, HTTP helper for temporary websocket token exchange. WebSocket v2, BSATN, reconnect handling, TLS support. Lynx adapter wraps NativeModules.WebSocketModule and base64-encodes binary messages; HTTP uses NativeModules.HttpModule or XHR. Needs hardening: custom adapter is demo-proven but should be tested across iOS, Android, Harmony, reconnect, token refresh, and network errors.
Generated bindings Generated typed tables, reducers, sum types, indexes, query builders. Generated typed APIs for tables, reducers, enums, and views. Generated bindings exist under lyxn-project/src/spacetimedb/module_bindings/. Covered for this app; generator workflow still needs to stay compatible with upstream schema changes.
React UI integration SpacetimeDBProvider, useSpacetimeDB, useTable, and useReducer. Not React; Flutter-facing reactive APIs such as streams/notifiers. Copies the React integration shape, but this Lynx app currently bypasses useTable for core counter state and uses SQL polling. Major gap: local useTable currently returns [[], false] instead of useSyncExternalStore(...), so it is not functionally complete.
Table subscriptions useTable(query) automatically subscribes and returns [rows, isReady]; raw subscriptionBuilder() supports SQL and typed queries. SubscribeMulti supports batching multiple SQL subscriptions. Raw subscriptionBuilder() exists and supports SQL strings, arrays, typed queries, and subscribeToAllTables(). Backend present, React hook incomplete: raw subscriptions exist, but the frontend hook layer needs upstream parity.
Local table cache Client cache exposes connection.db.table.iter(), count(), index access, and row callbacks. Table cache with iteration and insert/update/delete streams. Client cache and row callbacks exist in src/sdk/table_cache.ts. Present but not fully trusted in app: current app uses REST SQL fallback for auth/session reads because table-cache timing was not reliable enough.
Reducer calls Typed connection.reducers.reducerName(...) and React useReducer(reducer). Typed reducer calls returning a Future result. Typed reducer accessors exist and are used by login/counter actions. Timing gap: app needed runReducerWithTimeout because reducer ack promises can timeout in Lynx even when server state changed.
Reducer event callbacks / flags Official docs describe reducer event callbacks and reducer flags such as success-notify behavior. Reducer results and optimistic/offline behavior are package features. Core reducer calls exist; parity for reducer event helper APIs and flags should be audited against upstream. Audit needed: the local port may lag upstream reducer event/flag ergonomics.
One-off query API Official SDK has protocol support in source; docs focus on subscriptions and client cache for app reads. Subscription API is documented; generated table/view access is the main read path. Source logs: Received OneOffQueryResult but SDK does not expose one-off query APIs yet. Gap: app had to add lyxn-project/src/spacetimedb/sqlRead.ts and call REST SQL directly.
Auth and token persistence Builder accepts withToken; onConnect returns identity and token for persistence. Auth utilities with token persistence support. Token is saved in sessionStore and passed back through withToken. Works, but fragile: stale local app auth needed extra server-session validation in this demo.
Server-defined views TypeScript SDK supports generated module bindings and query/table style access; current docs emphasize tables/queries more than views. Explicitly documents generated typed views with the same cache/stream APIs as tables. No validated view usage in the Lynx demo. Unknown / unproven: needs a view module fixture and generated binding test before claiming support.
Offline and optimistic mutations Not a primary documented feature of the official React SDK. Offline mutation queue and optimistic updates are advertised features. No offline mutation queue or optimistic rollback layer. Gap vs Dart, but not necessarily a gap vs official React.
Reconnect lifecycle Connection manager retains/releases shared connections and exposes active/error state through React context. Reconnect handling is documented as part of the Dart connection layer. Connection manager exists, but the Lynx native adapter/reconnect path has not been fully validated. Needs tests: app-level reconnect, token reuse, dropped socket, and resumed subscriptions should be covered.
Production readiness for full frontend work Best fit for web React apps because it is official and current. Best fit for Flutter/Dart apps, especially if offline-first behavior is required. Best fit today for controlled Lynx demos and narrow app flows. Not complete yet: fix React hook parity, one-off query support, reducer ack behavior, session consistency, reconnection, and cross-platform native transport tests.

Sources checked: official TypeScript React docs at spacetimedb.com/docs/clients/typescript, official React quickstart at spacetimedb.com/docs/quickstarts/react, official source files under clockworklabs/SpacetimeDB/crates/bindings-typescript/src/react, Dart package docs at pub.dev/packages/spacetimedb, and local files under spacetimedb-lynx/src and lyxn-project/src/spacetimedb.
